add_trajectory(t, *args, extend_to=3, target=None, correct_start=False)

Add trajectory to the set from which PVPs are computed

Pass the time series, and any number of positional series. For example in dim3 with x, y, z, you would call (with defaults kwargs)

pvp.add_trajectory(t, x, y, z, extend_to = 3, target = None, correct_start = False)

You control the duration of the PVP (e.g. how far in time trajectories are extended). You also need to specify the target location for each trajectory. You can optionally synchronize the trajectories by pre-processing them (correct_start). Currently, a simple thresholding rule takes care of this synchronization.

Parameters

  • t (numpy.array like) – time series

  • args (numpy.array like) – positional series

  • extend_to (int, optional) – minimal PVP duration, defaults to 3

  • target (iterable, optional) – target location, defaults to None. If None, will use the null vector as target.

  • correct_start (bool, optional) – whether to correct the location of the start of the movement for synchronization, defaults to False

plot_std_profiles(ax=None, fit=True, prof_kwargs=None, fit_kwargs=None)

Plots the standard deviation profiles on the provided axis. If not provided, will create a new figure from scratch. If fit is True, will also compute the spline fit to the second and third phase. Keyword arguments to the std plotter (prof_kwargs) and to the fit plotter (fit_kwargs) can also be given.

Note

You should have fitted the profiles prior to plotting them.

Parameters

  • ax (plt.axis, optional) – axis on which to draw, defaults to None. If None, creates a new figure and axis to draw on.

  • fit (bool, optional) – whether to plot the spline fit, defaults to True

  • prof_kwargs (dict, optional) – keyword arguments are passed to plotter for the standard deviation profile, defaults to None

  • fit_kwargs (dict, optional) – keyword arguments are passed to plotter for the spline fit, defaults to None
